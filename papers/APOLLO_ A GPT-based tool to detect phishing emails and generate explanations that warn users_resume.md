## Summary — *APOLLO: A GPT-based tool to detect phishing emails and generate explanations that warn users*

This paper presents **APOLLO**, a tool that uses **GPT-4o** to detect phishing emails and generate warning explanations for users. The main idea is not only to classify an email as phishing or legitimate, but also to explain **why** the email may be dangerous in simple language that non-expert users can understand. 

## Main problem

The authors argue that phishing remains highly effective because it attacks both the **technical side** and the **human side** of security. Existing systems may detect suspicious emails or websites, but users often ignore warnings because they are too generic, passive, repetitive or difficult to understand.

The paper highlights that warnings are often ineffective because users do not understand the risk, do not trust the warning, or become used to seeing similar alerts repeatedly. This is known as the **habituation effect**. 

## What APOLLO does

APOLLO has two main functions:

1. **Classifies an email** as phishing or legitimate.
2. **Generates an explanation** telling the user why the email is suspicious.

The tool was developed in **Python** and uses **OpenAI’s GPT-4o**. Its architecture has three main modules:

| Module           | Function                                                                            |
| ---------------- | ----------------------------------------------------------------------------------- |
| **Preprocessor** | Extracts subject, headers, body and URLs from the email                             |
| **URL enricher** | Gets extra information about URLs using services like VirusTotal and BigDataCloud   |
| **LLM prompter** | Sends the processed information to GPT-4o and receives classification + explanation |

The architecture diagram on page 6 shows the flow: email input → preprocessing → URL enrichment → GPT-4o → classification and warning explanation. 

## How the tool works

First, APOLLO processes the email by extracting:

* email subject;
* headers;
* body text;
* URLs;
* phone numbers;
* email addresses.

Then, if the email contains URLs, the tool enriches the information using external services. For example, **VirusTotal** gives information about whether the URL is considered harmless, undetected or malicious. This extra information helps GPT-4o make a better decision. 

After that, GPT-4o receives a structured prompt asking it to output:

* the classification label: phishing or legitimate;
* the probability of being phishing;
* persuasion techniques used in the email;
* suspicious or legitimate features;
* a short explanation for the user.

## Main results: classification performance

The authors tested APOLLO with a dataset of **4000 emails**, half phishing and half legitimate. Without URL information, GPT-4o achieved:

* **97.4% accuracy**;
* **98.5% recall**;
* **96.4% precision**;
* **97.4% F1-score**.

When reliable URL information from external services was included, performance improved to almost perfect results, reaching around **99.9% accuracy** in some conditions. 

However, the paper also shows an important limitation: if the external URL information is wrong or uncertain, it can confuse the model and reduce performance. This means that integrating external threat intelligence is useful, but only when that information is reliable. 

## User study

The authors also evaluated how users perceived the warnings generated by APOLLO. They conducted a study with **20 participants**, comparing APOLLO-generated warnings with warnings from:

* Chrome;
* Firefox;
* Edge;
* a manually written warning.

The APOLLO warnings were generally perceived as:

* more understandable;
* more interesting;
* more trustworthy;
* high quality overall.

The warnings explained specific phishing indicators, such as:

* URL being an IP address;
* top-level domain placed strangely;
* mismatch between shown link and real link;
* newly created domain.

## Important contribution

The biggest contribution of this paper is that it combines **technical phishing detection** with **human-centered explanations**.

Most phishing detection systems only classify an email. APOLLO goes further by explaining the risk to the user. This is important because, in many cases, the final decision still depends on the user.

For your topic, this is very useful because it supports the idea that modern security filters should not only block threats automatically, but also help users understand suspicious content.

## Limitations

The paper identifies several limitations:

* the user study only had 20 participants;
* the generated explanations depend heavily on prompt quality;
* LLMs can hallucinate or generate explanations that are not fully grounded;
* only GPT-4o was tested in the final classification evaluation;
* the tool only considered the first URL in the email;
* external services like VirusTotal can improve results, but wrong information can damage classification performance;
* the study measured user perception, not real-world click behavior.

This paper is very relevant for **“Modern Phishing vs. Security Filters”** because it shows a modern direction for phishing defense: using LLMs not only as classifiers, but also as explanation generators.

You can use this paper to support these arguments in your article:

* traditional filters are not enough;
* phishing detection must consider both technical and human factors;
* LLMs can improve phishing detection;
* explanations can make warnings more useful and trustworthy;
* external threat intelligence can improve classification, but must be reliable;
* there is a trade-off between accuracy, user trust and system reliability.

## Final takeaway

The main conclusion is that **LLMs are promising tools for phishing defense**, especially when combined with external threat intelligence and user-friendly explanations. APOLLO is presented as a proof of concept showing that GPT-based systems can both detect phishing emails with high accuracy and generate warnings that users perceive as understandable and trustworthy.
